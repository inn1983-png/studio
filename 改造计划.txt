# Txtovideo Studio 长期改造规划

仓库：inn1983-png/studio
当前定位：基于 ai-moive-studio / AICON 改造的小说短剧工作站
最终目标：把它改造成一个面向个人生产的 AI 短剧工作台，用于小说/文案 → 剧本 → 角色/场景/道具 → 分镜 → 图片提示词 → 视频提示词 → 导出 → 接 ComfyUI / RunningHub / 剪映。

重要原则：
1. 不推倒重来。
2. 不追求一开始就做成万能平台。
3. 当前项目作为 Txtovideo Studio V1。
4. 先跑通生产链路，再做画布增强。
5. 所有改动必须小步提交，每次提交必须能启动、能构建、能回退。
6. 不删除原有核心功能，先隐藏无关入口。
7. 先做导出型工作台，再做自动生成型工作台。
8. 先保证数据结构稳定，再接复杂模型和 ComfyUI。
9. 不要把九宫格方案作为默认主线。
10. 默认主线为：每个分镜生成一张独立图片提示词。

Phase 2：短剧主流程 MVP
==================================================

目标：
先做一个不依赖复杂画布的“线性短剧工作台”。

页面路径建议：
/txtovideo/projects/:projectId/workbench

页面布局：
左侧：步骤导航
中间：当前步骤编辑区
右侧：结果预览 / JSON 预览 / 操作按钮

步骤如下：
1. 原文输入
2. 剧本改编
3. 角色资产
4. 场景资产
5. 道具资产
6. 分镜生成
7. 图片提示词
8. 视频提示词
9. 质量检查
10. 导出

每一步都必须：
- 可查看输入
- 可查看输出
- 可手动编辑输出
- 可重新生成
- 可保存
- 可进入下一步

不要一开始就自动跑完全流程。
V1 默认用户手动点击每一步运行。

具体任务：

1. 新建短剧工作台入口
- 项目详情页增加按钮：进入短剧工作台
- 导航增加：短剧工作台

2. 原文输入页
支持：
- 粘贴文本
- 上传 TXT
- 显示字数
- 显示预计分镜数量
- 设置时代
- 设置风格
- 设置目标平台
- 设置画面比例

默认参数：
- 画面比例：9:16
- 风格：古风真人写实
- 平台：抖音/视频号
- 单镜头时长：8-12 秒

3. 剧本改编页
输出格式：
【OS】
【角色名】
【留白】

要求：
- 支持一键生成
- 支持手动编辑
- 支持保存版本
- 支持对比版本

4. 角色资产页
功能：
- 从剧本提取角色
- 显示角色卡片
- 支持编辑角色名称、性别、年龄、身份、外貌、服装
- 支持生成角色定妆照提示词
- 支持锁定角色

5. 场景资产页
功能：
- 从剧本提取场景
- 显示场景卡片
- 支持编辑场景名称、时代、光线、氛围、道具
- 支持生成场景图提示词
- 支持锁定场景

6. 道具资产页
功能：
- 提取重要道具
- 支持编辑
- 支持标记是否必须出现在画面中

7. 分镜生成页
输出数组，每项必须包含：
{
  "theme": "两个汉字",
  "cap": "原文连续片段",
  "desc_promopt": "画面描述"
}

要求：
- 允许手动编辑 JSON
- 提供表格视图
- 提供逐条预览
- 检查 cap 是否为空
- 检查 desc_promopt 是否为空
- 检查 theme 是否为两个汉字

8. 图片提示词页
每个分镜生成：
- positive_prompt
- negative_prompt

默认负向词必须包含：
现代物品、现代建筑、现代服装、手机、电脑、电灯、眼镜、手表、拉链、塑料、欧美人、卡通、动漫、3D渲染、游戏风、低质量、模糊、畸形、多肢体、多人错误、性别错误

9. 视频提示词页
每个分镜生成：
- duration_seconds
- camera_motion
- character_motion
- scene_motion
- prompt
- avoid

LTX 默认控制原则：
- 动作轻
- 镜头慢
- 角色不大幅移动
- 场景不跳变
- 光线不突变
- 避免快速转身
- 避免多人复杂交互
- 避免人物出画再入画

10. 导出页
导出 ZIP 包，内容见 Phase 4。

验收标准：
- 用户可以从一段文本完整走到导出
- 每一步可以手动保存
- 不要求真实生成图片/视频
- 重点是结构化内容稳定

Phase 3：提示词模板体系重建
==================================================

目标：
把提示词从散落文档，升级成系统内可维护的模板。

目录建议：
backend/src/prompts/txtovideo/
backend/docs/prompts/txtovideo/

7. video_prompt_ltx_v1.md
8. video_prompt_seedance_v1.md
9. quality_score_v1.md
10. rewrite_fix_v1.md

提示词必须支持变量：
- source_text
- script_text
- characters
- scenes
- props
- era
- style
- platform
- aspect_ratio
- shot_count_min
- shot_count_max
- duration_seconds
- negative_rules

核心提示词规则：

1. 剧本改编
输出：
【OS】
【角色名】
【留白】

风格：
- 短句
- 口播友好
- 电视剧感
- 节奏清晰
- 对白尊重原文
- 不要过度文艺
- 不要大段解释

2. 角色提取
必须输出 JSON。
必须明确：
- name
- gender
- age
- identity
- appearance
- costume
- stable_prompt

强规则：
- 一旦确认性别，后续必须一致
- 主角不能被错误生成年龄
- 不能把一个人拆成两个人

3. 场景提取
必须输出 JSON。
必须强调：
- 中国古代语境
- 场景可视化
- 光线
- 道具
- 空间结构
- 时代一致性

4. 分镜生成
必须输出：
[
  {
    "theme": "两个汉字",
    "cap": "原文连续片段",
    "desc_promopt": "画面描述"
  }
]

硬规则：
- cap 必须严格取自原文连续文本
- 不允许改写 cap
- 不允许重组 cap
- 不允许创造剧情
- 第一个分镜默认不出现人物
- 角色必须来自角色库
- 未识别人物统一称为“龙套”
- 按文本顺序生成
- 1200-2500 字文案，分镜数量 50-100
- desc_promopt 要能直接生成图像

5. 图片提示词
正向风格默认：
电视剧电影真人写实风格，东方人物，中国古代服饰，真实纹理，自然颜色，电影级摄影，浅景深，真实光照，古代建筑，画面清晰，情绪明确。

负向默认：
现代、21世纪、现代建筑、现代服饰、手机、电脑、电灯、眼镜、手表、拉链、塑料、欧美人、卡通、动漫、3D、游戏风、低质、模糊、畸形、多肢体、多人错误、性别错误。

6. 视频提示词
默认适配 LTX：
- 保持人物身份一致
- 保持场景一致
- 保持服装一致
- 镜头缓慢推进/轻微横移
- 人物轻微动作
- 烛火/衣袖/发丝/雪/雨/尘埃可轻微运动
- 禁止快速动作
- 禁止跳切
- 禁止换脸
- 禁止场景突变

验收标准：
- 所有模板集中管理
- 前端可选择模板版本
- 生成结果保存 prompt_used
- 方便后续迭代 V2/V3/V4

Phase 5：任务状态机与步骤重试
==================================================

目标：
解决长任务执行顺序混乱、上游未完成下游先跑、失败后无法定位的问题。

新增统一步骤状态：
- pending
- running
- success
- failed
- skipped
- waiting_dependency

每个步骤保存：
- step_name
- input_hash
- input_snapshot
- output_snapshot
- status
- error_message
- started_at
- finished_at
- retry_count
- model_used
- prompt_used

短剧流程步骤：
1. script_adapt
2. character_extract
3. scene_extract
4. prop_extract
5. storyboard_generate
6. image_prompt_generate
7. video_prompt_generate
8. quality_score
9. export_package

执行规则：
- 下游必须等待上游 success
- 上游 failed 时下游进入 waiting_dependency
- 用户可以重试当前步骤
- 用户修改上游内容后，下游标记为 stale
- stale 的下游需要提示用户重新生成

前端显示：
- 每一步状态图标
- 成功/失败/等待/过期
- 错误信息可展开
- 支持重试按钮

验收标准：
- 不会出现图像没生成完就跑视频提示词的问题
- 每一步失败能看到错误
- 每一步能独立重试

Phase 8：音频驱动版本
==================================================

目标：
支持用户当前的音频驱动视频流程。

音频驱动模式：
- 一章文本生成一段完整音频
- 工作流按 12 秒切段
- 每段生成对应视频
- 最后拼接成完整 MV/短剧片段

新增模式：
workflow_mode:
- txtovideo_storyboard
- audio_driven_video

音频驱动流程：
1. 输入文本
2. 生成 OS/对白/留白剧本
3. 生成完整音频文本
4. 生成 TTS / 外部音频
5. 上传音频
6. 自动切分 12 秒段落
7. 为每段生成画面描述
8. 为每段生成图片提示词
9. 为每段生成视频提示词
10. 导出音频驱动包

音频驱动导出：
- audio_script.txt
- audio_segments.json
- segment_image_prompts.json
- segment_video_prompts.json
- subtitles.srt
- manifest.json

重要规则：
- 不额外强制复杂时间轴
- 以音频切段为主
- 每段默认 12 秒
- 支持用户调整每段文本和提示词
- 支持 OS / 对白 / 留白结构

验收标准：
- 用户可以上传一段音频
- 系统能按 12 秒切段
- 每段能生成对应视频提示词
- 能导出给现有工作流使用

==================================================
Phase 9：质量评分与自动修复
==================================================

目标：
先做文本和 JSON 层面的评分，不直接做视频评分。

评分对象：
1. 剧本
2. 角色资产
3. 场景资产
4. 分镜 JSON
5. 图片提示词
6. 视频提示词

评分维度：

剧本评分：
- 是否保留原文核心信息
- 是否有 OS / 对白 / 留白
- 是否口播友好
- 是否节奏清晰
- 是否过度改写

角色评分：
- 是否性别明确
- 是否年龄合理
- 是否服装符合时代
- 是否存在角色重复
- 是否存在现代元素

场景评分：
- 是否符合时代
- 是否可视化
- 是否光线明确
- 是否道具合理
- 是否现代元素污染

分镜评分：
- cap 是否连续取自原文
- theme 是否两个汉字
- desc_promopt 是否为空
- 角色是否来自角色库
- 第一镜是否避免人物
- 数量是否符合要求

图片提示词评分：
- 是否真人写实
- 是否古风
- 是否避免现代元素
- 是否避免欧美人
- 是否避免卡通/3D
- 人数是否明确
- 性别是否明确

视频提示词评分：
- 是否动作过大
- 是否镜头过快
- 是否容易导致换脸
- 是否场景跳变
- 是否符合 LTX 稳定策略

自动修复：
- 评分低于阈值时，生成修复建议
- 用户点击“应用修复”
- 保留修复前版本
- 不自动覆盖用户手动编辑内容

验收标准：
- 能对 storyboard.json 打分
- 能指出具体第几条有问题
- 能生成修复建议
- 能应用修复并保存新版本
